## 一、專題主題與動機

校園裡常常需要臨時借用物品：突然下雨需要雨傘、手機沒電需要行動電源、上台簡報需要 HDMI 線。
但這些公用物品常常「不知道在哪借」「借了沒人記錄」「借走忘了還」。

我們做了一個 **線上物品借用系統**，讓師生可以：

- 上網查詢有哪些物品可借、還剩幾個
- 一鍵借用並設定歸還期限
- 隨時查看自己借了什麼、有沒有逾期
- 管理員（如學務處／系辦）統一維護物品與追蹤所有借用紀錄

這類系統做好後可以「真的流傳下去」被學弟妹繼續使用，符合本次專題「資訊恆久遠，一站永流傳」的主旨。

## 二、使用技術

| 層級 | 技術 | 說明 |
| --- | --- | --- |
| 前端 | **Vue 3 + Vite** | 元件化畫面、Vue Router 切換頁面 |
| 後端 | **Express** | 提供 REST API |
| 資料庫 | **SQLite**（Node 內建 `node:sqlite`） | 免安裝、單一檔案 `borrow.db` |
| 語言 | HTML / CSS / JavaScript | 符合作業限制 |

> 💡 為什麼用 `node:sqlite`？Node.js 22.5 之後內建了 SQLite，**不必再安裝會編譯失敗的套件（如 better-sqlite3）**，`npm install` 只需要 express，跨電腦最不容易出錯。

架構圖：

```
[瀏覽器 Vue 前端 :5173]  --/api 代理-->  [Express 後端 :3000]  --->  [SQLite borrow.db]
```

## 四、資料庫設計

共四張表：

```sql
users(id, username, password, name, role)        -- role: user / admin
items(id, name, category, description, emoji,
      total_qty, available_qty)                   -- 總數與目前可借數
loans(id, item_id, user_id, qty,
      borrowed_at, due_at, returned_at, status)    -- status: borrowed / returned
tokens(token, user_id)                            -- 登入憑證
```

設計重點：

1. **庫存控制**：借出時 `available_qty - 數量`，歸還時加回，避免超借。
2. **逾期判斷**：不另存欄位，查詢時用「未歸還且 `due_at` < 現在」即時算出 `overdue`。
3. **密碼安全**：用 Node 內建 `crypto.scryptSync` 加鹽雜湊，不存明碼。

> 📷 _建議截圖：用 DB 工具或 `SELECT * FROM items` 看資料表內容_

## 五、後端實作（server）

### 1. 建立資料庫與資料表（db.js）

使用 `node:sqlite` 的 `DatabaseSync` 開啟資料庫檔，並用 `CREATE TABLE IF NOT EXISTS` 建表。

```js
const { DatabaseSync } = require('node:sqlite');
const db = new DatabaseSync('borrow.db');
db.exec(`CREATE TABLE IF NOT EXISTS items ( ... )`);
```

### 2. 範例資料（seed.js）

第一次執行 `npm run seed`，會建立管理員 / 學生帳號與 10 項範例物品。

### 3. API 路由（index.js）

以中介層 `requireAuth` / `requireAdmin` 控制權限；借用與歸還用**交易（transaction）** 確保庫存與紀錄同時更新成功。

借用的核心邏輯：

```js
app.post('/api/loans', requireAuth, (req, res) => {
  // 1. 檢查可借數量是否足夠
  // 2. items.available_qty 減少
  // 3. 新增一筆 loans，due_at = 現在 + 借用天數
});
```

> 📷 _建議截圖：終端機執行 `npm start` 顯示「🚀 API 伺服器已啟動」_

## 六、前端實作（client）

### 1. 登入狀態管理（store.js）

用 Vue 的 `reactive` 保存 token 與使用者資訊，存進 `localStorage`，重新整理也不會登出。
並包裝一個 `api()` 函式，自動帶上 `Authorization` 標頭、自動處理錯誤訊息。

### 2. 路由與權限（router.js）

```js
{ path: '/my-loans', component: MyLoans, meta: { auth: true } }   // 需登入
{ path: '/admin',    component: Admin,   meta: { admin: true } }  // 需管理員
```

用 `router.beforeEach` 守衛：沒登入跳轉登入頁、不是管理員擋下後台。

### 3. 各頁面（views）

- **Home.vue**：物品卡片牆、搜尋框、分類下拉、借用彈窗。
- **Login.vue / Register.vue**：登入與註冊。
- **MyLoans.vue**：我的借用表格，可按「歸還」。
- **Admin.vue**：物品管理（新增／編輯／刪除）與全校借用紀錄。

## 八、遇到的問題與解決

> 這段請依你實際情況補充，例如：

- **資料庫套件安裝失敗**：改用 Node 內建 `node:sqlite`，免編譯就解決了。
- **前端呼叫後端跨域（CORS）**：後端加上 `cors()`，並在 Vite 設定 `/api` 代理。
- **重新整理後登入消失**：把 token 存進 `localStorage`。
